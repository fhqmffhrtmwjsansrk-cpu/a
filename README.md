# 교실 위치별 공기질 측정 프로젝트

교실 안에서 **어느 자리가 공부하기 좋은지**를 데이터로 알아보는 프로젝트입니다.
Raspberry Pi Pico 2W와 가스센서 2대로 교실을 왼쪽(입구측)/오른쪽(창가측) 두 구역으로 나누어
상대 공기질을 측정하고, 결과를 웹 대시보드로 시각화했습니다.

## 측정 결과 요약

| 구역 | 상대 공기질 지수 | 판정 |
|---|---|---|
| 왼쪽 (입구측) | 8,944 | 약간 나쁨 |
| 오른쪽 (창가측) | 8,493 | 약간 좋음 |

**결론: 창가측이 약간 좋은 경향이 관찰되었으나, 유의한 차이로 단정할 수 없음.**

두 구역의 차이(약 450)가 센서 오차 요인(예열 드리프트 약 1,900, 개체차 변동 약 900)보다
작았기 때문입니다. 차이가 없다는 것도 측정으로 확인한 결과입니다.

## 실험 설계: 교차설계 (Crossover Design)

이 프로젝트의 핵심입니다. MQ 계열 센서는 개체차가 매우 커서
(실측: 두 센서가 같은 공기에서 **4,075** 차이) 그냥 두 곳에 놓고 재면
"어느 구역이 나쁜가"가 아니라 "어느 센서가 높게 읽는가"를 측정하게 됩니다.

이를 해결하기 위해 측정 중간에 두 센서의 자리를 서로 맞바꿨습니다.

```
phase 0 (4분) : 두 센서를 중앙에 나란히  → 개체차 직접 측정
phase 1 (8분) : 센서 A = 입구측 / 센서 B = 창가측
phase 2 (8분) : 센서 A = 창가측 / 센서 B = 입구측  (맞바꿈)
```

각 센서가 두 구역을 모두 경험하므로, 구역별 평균을 낼 때
**센서 개체차와 시간에 따른 변화가 함께 상쇄**됩니다.

```
입구측 효과 = (A의 phase1 + B의 phase2) / 2
창가측 효과 = (B의 phase1 + A의 phase2) / 2
```

추가로 각 위치 구간의 앞 2분은 이동 직후 안정화 시간으로 보고 분석에서 제외했습니다.

## 파일 구성

```
├── main_mq.py        # Pico 2W 측정 로거 (MicroPython)
├── analyze_mq.py     # PC 분석 스크립트 (Python + pandas)
├── dashboard.html    # 결과 시각화 웹 대시보드
├── scd30.py          # SCD30 드라이버 (초기 계획, 하드웨어 이슈로 미사용)
├── scd30_diag.py     # SCD30 I2C 진단 도구
└── main_auto.py      # SCD30 2대용 로거 (초기 계획)
```

## 하드웨어

- Raspberry Pi Pico 2W × 2
- MQ 계열 가스센서 × 2

배선 (Pico마다):

| 센서 핀 | Pico 핀 |
|---|---|
| VCC | VBUS (40번, 5V) — 히터가 5V 필요, 3V3 아님 |
| GND | GND |
| AOUT | GP26 (ADC0) |

## 사용 방법

### 1. 측정 (Pico 2대)

`main_mq.py`를 두 Pico에 올리고, 한 대만 `DEVICE_ID = "B"`로 수정합니다.
두 장치를 거의 동시에 실행하면 3분 예열 후 측정이 시작되고,
LED 깜빡임(phase 번호만큼)이 센서를 옮길 타이밍을 알려줍니다.
결과는 각 Pico의 플래시에 `log_mq_A.csv` / `log_mq_B.csv`로 저장됩니다.

### 2. 분석 (PC)

```bash
pip install pandas matplotlib
python analyze_mq.py   # log_mq_A.csv, log_mq_B.csv 를 같은 폴더에 두고 실행
```

센서 개체차, 구역별 요약, 교차설계 보정 결과, 시계열 그래프(`result_mq.png`)를 출력합니다.

### 3. 대시보드

`dashboard.html`을 브라우저에서 열면 교실 평면도 위에서 구역별 결과를 확인할 수 있습니다.

## 한계

- **MQ 센서는 CO₂ 절대 농도(ppm)를 측정할 수 없습니다.** 반도체식 센서라 여러 가스에
  복합적으로 반응하므로, 모든 수치는 상대 비교 지수입니다.
- 히터 예열 3분이 부족해 측정 중 하강 드리프트(약 1,900)가 발생했습니다.
  두 센서가 자리를 맞바꾼 뒤에도 동반 하락한 것이 그 증거입니다.
- 측정 1회, 20분, 2개 지점으로 표본이 제한적이며, 재실 인원 등 교란 변수를 통제하지 못했습니다.

## 개선 방향

- 측정 시작 전 **30분 이상 사전 예열**로 드리프트 제거
- NDIR 방식 센서(SCD30 등) 사용 시 CO₂ 절대값(ppm) 측정 가능
  — 원래 SCD30으로 계획했으나 I2C 통신 불안정으로 전환 (진단 도구는 `scd30_diag.py` 참고)
- 요일/시간대를 바꿔 반복 측정하여 재현성 확인
- 온습도 센서를 추가해 온열 쾌적성(에어컨 직바람 여부 등)까지 종합 평가

## 배운 점

측정된 오차 요인의 크기 순위: **센서 개체차(4,075) > 예열 드리프트(1,900) > 구역 간 차이(450)**

"무엇을 재고 싶은가"보다 "무엇이 측정을 방해하는가"를 먼저 정량화해야
결과를 올바르게 해석할 수 있다는 것이 이 프로젝트의 가장 큰 결론입니다.
